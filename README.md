# НеоФин ИИ (NeoFin AI)

> Гибридная ИИ-система финансового анализа предприятий

---

## Что делает система

NeoFin AI автоматически анализирует финансовую отчётность компаний: извлекает данные из PDF (включая сканы), вычисляет 13 коэффициентов по четырём группам, формирует интегральный скоринг 0–100 и генерирует обоснованные рекомендации через языковые модели.

**Для кого:** финансовые директора, аналитики, инвесторы, аудиторы — все, кто принимает решения на основе финансовой отчётности и ценит скорость и прозрачность анализа.

**Какую проблему решает:** ручной анализ PDF-отчётов занимает часы и не даёт оценки надёжности данных. NeoFin AI выполняет полный цикл за секунды и объясняет каждое извлечённое число.

---

## Гибридная архитектура

Система работает на двух независимых уровнях:

**Уровень 1 — детерминированный (всегда активен)**
Извлечение данных → фильтрация по Confidence Score → расчёт коэффициентов → интегральный скоринг. Воспроизводимо, объяснимо, не зависит от AI-провайдеров.

**Уровень 2 — вероятностный (AI-слой)**
NLP-анализ рисков и генерация рекомендаций через языковые модели. При недоступности провайдера система продолжает работу — числовой анализ не прерывается.

```
PDF
 │
 ▼
Extractor ──► Confidence Score + Extraction Metadata
 │                    │
 ▼                    ▼
Filter (порог 0.5)   Explainability UI (🟢🟡🔴)
 │
 ▼
Ratios (13 коэффициентов)
 │
 ▼
Scoring (0–100, risk_level, factors)
 │
 ▼
AI Pipeline: GigaChat → DeepSeek → Ollama → rule-based fallback
 │
 ▼
React / Mantine UI  ←→  FastAPI  ←→  PostgreSQL
```

## Ключевые возможности

**Извлечение и анализ данных**
- Обрабатывает текстовые PDF, таблицы и сканы (OCR через pytesseract)
- Вычисляет 13 коэффициентов: ликвидность, рентабельность, финансовая устойчивость, деловая активность
- Формирует интегральный скоринг 0–100 с уровнем риска и факторами влияния

**Confidence Score и Explainability**
- Присваивает каждому показателю оценку надёжности [0.0–1.0] на основе метода извлечения
- Исключает ненадёжные данные из расчётов автоматически (порог настраивается)
- Отображает источник и метод извлечения в tooltip для каждого показателя
- Показывает сводку: «Извлечено надёжно: N из 15 показателей»

**NLP и рекомендации**
- Выявляет финансовые риски и ключевые факторы через языковые модели
- Генерирует рекомендации с явными ссылками на конкретные метрики
- Fallback chain: GigaChat → DeepSeek → Ollama → встроенные правила

**Многопериодный анализ**
- Сравнивает до 5 отчётных периодов в одной сессии
- Строит интерактивный график динамики коэффициентов (TrendChart)
- Автоматически сортирует периоды хронологически, выявляет аномальные изменения

**Надёжность и независимость**
- Offline-ready: полный числовой анализ без интернета через локальную модель Ollama
- История анализов с постраничным просмотром и детализацией

## Confidence Score: как это работает

Каждый извлечённый показатель получает оценку на основе метода извлечения:

| Метод | Тип источника | Confidence |
|---|---|:---:|
| Точное совпадение ключевого слова в таблице | `table_exact` | 0.9 |
| Частичное совпадение в таблице | `table_partial` | 0.7 |
| Извлечение через regex из текста | `text_regex` | 0.5 |
| Производный расчёт (например, обязательства = активы − капитал) | `derived` | 0.3 |

Показатели ниже `CONFIDENCE_THRESHOLD` (по умолчанию 0.5) автоматически исключаются из расчёта коэффициентов. В UI каждый показатель маркируется цветом: 🟢 > 0.8 / 🟡 0.5–0.8 / 🔴 < 0.5.

## Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-org/neofin-ai.git && cd neofin-ai

# 2. Создать файл окружения
cp .env.example .env
# Заполнить DATABASE_URL, API_KEY; опционально — ключи AI-провайдеров

# 3. Собрать и запустить
docker-compose up --build

# 4. Миграции применяются автоматически при старте
# Вручную: docker-compose exec backend alembic upgrade head

# 5. Открыть приложение: http://localhost
```

---

## Запуск

**Dev:**
```bash
docker-compose up --build
```
Поднимает backend (FastAPI), frontend (Vite), PostgreSQL, Ollama.

**Production:**
```bash
./scripts/start-prod.sh
```
Проверяет `.env`, запускает `docker-compose.prod.yml`, применяет миграции. Доступно на порту 80.

## Пример использования

**Анализ одного отчёта:**
```bash
# Загрузить PDF
curl -X POST http://localhost/api/upload \
  -H "X-API-Key: your_key" \
  -F "file=@report_2023.pdf"
# → {"task_id": "abc-123", "status": "processing"}

# Получить результат
curl http://localhost/api/result/abc-123 -H "X-API-Key: your_key"
# → {"status": "completed", "data": {"score": {...}, "ratios": {...}, "extraction_metadata": {...}}}
```

**Многопериодный анализ:**
```bash
# Запустить сессию
curl -X POST http://localhost/api/multi-analysis \
  -H "X-API-Key: your_key" \
  -H "Content-Type: application/json" \
  -d '{"periods": [{"period_label": "2021"}, {"period_label": "2022"}, {"period_label": "2023"}]}'
# → {"session_id": "xyz-456", "status": "processing"}

# Отслеживать прогресс
curl http://localhost/api/multi-analysis/xyz-456 -H "X-API-Key: your_key"
# → {"status": "processing", "progress": {"completed": 2, "total": 3}}
# → {"status": "completed", "periods": [...]}
```

**Просмотр динамики в UI:** вкладка **«Динамика»** на странице отчёта — интерактивный TrendChart с выбором коэффициентов, индикаторами тренда (↑↓) и маркерами аномалий (⚠).

## Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `DATABASE_URL` | — | PostgreSQL connection string (обязательно) |
| `API_KEY` | — | Ключ доступа к API (обязательно) |
| `CONFIDENCE_THRESHOLD` | `0.5` | Порог надёжности: показатели ниже исключаются из расчётов |
| `GIGACHAT_CREDENTIALS` | — | Ключ GigaChat API |
| `HF_API_KEY` | — | Ключ DeepSeek API |
| `USE_LOCAL_LLM` | `false` | Использовать Ollama (offline-режим) |
| `DEMO_MODE` | `0` | Маскировать числовые данные в ответах API |

Полное описание всех переменных — в `docs/CONFIGURATION.md`.
