# Операторская карточка показа

Краткий сценарий для выступления без автора изменений.

## 1. Порядок экранов (5-7 минут)

1. Главный экран и загрузка PDF
2. Статус обработки (real-time обновления)
3. Итоговый отчет:
- `score`, `risk_level`
- ключевые метрики
- блок надежности извлечения
- факторы и рекомендации
4. История анализов (`/analyses`)
5. Multi-period динамика (TrendChart)

## 2. Какие файлы показывать

- `text_single`: годовой текстовый отчет
- `scanned_single`: OCR-heavy скан
- `multi_period_magnit`: три периода для тренда

Сценарии и ожидания зафиксированы в `tests/data/demo_manifest.json`.

## 3. Что проговаривать при low-confidence

- Система явно показывает источник и уровень уверенности метрик
- Показатели с низкой уверенностью не скрываются, а помечаются для оператора
- Числовой контур остается воспроизводимым даже при недоступности LLM

## 4. Fallback за один шаг

Если публичный контур нестабилен:
1. Перейти на локальный primary path
2. Запустить:
```bash
python scripts/demo_smoke.py --base-url http://localhost --api-prefix /api --api-key <API_KEY>
```
3. Продолжить показ тем же порядком экранов
