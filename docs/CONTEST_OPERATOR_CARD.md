# Операторская карточка показа

Краткий сценарий для выступления без автора изменений.

## 1. Порядок экранов

1. Главная страница `НеоФин.Документы`
2. Выбор AI-провайдера рядом с загрузкой
3. Загрузка PDF и этапы обработки
4. Итоговый отчёт
5. История анализов
6. При необходимости — многопериодный сценарий

## 2. Что обязательно показать в отчёте

- итоговый `score`;
- `risk_level`;
- ключевые метрики;
- блок надёжности извлечения;
- badges benchmark’ов;
- блок `Как рассчитано`;
- AI-состояние или честный deterministic fallback.

## 3. Что проговаривать

### Про продукт

- `НеоФин.Документы` — первый модуль экосистемы `НеоФин.Контур`.
- Сервис анализирует финансовые документы, а не только “генерирует текст”.

### Про score

- итоговый score сопровождается методикой;
- интерфейс показывает, какой benchmark выбран;
- для retail-кейсов возможен `retail_demo`, для общих — `generic`;
- для interim-периодов система явно показывает `annualized_q1` или `annualized_h1`, если это применимо.

### Про AI

- AI-контур не скрывает своё состояние;
- если narrative пустой или провайдер завершился с ошибкой, интерфейс честно показывает deterministic fallback;
- числовой результат остаётся доступным даже без AI.

## 4. Какие файлы использовать

- `text_single`
- `scanned_single`
- `multi_period_magnit`

Источник — `tests/data/demo_manifest.json`.

## 5. Что делать при low-confidence

- не скрывать это;
- показать `extraction_metadata`;
- объяснить, что confidence влияет на расчёт, а не просто рисует красивый результат.

## 6. Что делать при сбое AI

Если AI-блок не дал содержательного narrative:

- не выдавать это за “падение всего продукта”;
- показать, что score, коэффициенты и методика всё равно доступны;
- при локальном показе при необходимости переключиться на `Ollama`.

## 7. Быстрый fallback

Если публичный контур нестабилен:

1. перейти на локальный контур;
2. открыть интерфейс `http://127.0.0.1/`;
3. выбрать `Ollama`;
4. повторить сценарий показа.
