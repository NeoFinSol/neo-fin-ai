# План реорганизации публичной документации

Документ зафиксирован до/в ходе выноса внутренних материалов из публичного GitHub-контура. Дата ориентира: 2026-04-20.

## Принципы

- **Публичное ядро** остаётся в репозитории и доступно на GitHub.
- **Архив конкурса** остаётся публичным, но изолирован под `docs/archive/contest/`.
- **Внутренние / служебные / стратегические** документы удаляются из отслеживаемого публичного дерева `docs/`; копия для владельца — в каталоге `docs/tech_docs_private/` (в `.gitignore` рядом с `.agent/`, не публикуется).
- Перенос в подпапку внутри того же публичного репозитория **не** считается мерой приватности для материалов, которые должны исчезнуть с GitHub.

## Спорные / ручное решение

| Вопрос | Рекомендация в рамках текущего запроса |
| --- | --- |
| Нужен ли `docs/ROADMAP.md` в публичном ядре | **Да** — текст актуален; включён в ядро и снят с gitignore для публичной видимости |
| Тесты, в строках которых фигурируют пути `docs/scoring_freeze_*.md` | Не менять код по задаче; строки остаются литеральными ссылками в сгенерированном тексте, файлы в `docs/` удалены — при необходимости синхронизировать формулировки тестов в отдельной задаче |

## Локальный контур сохранения

- Каталог: **`docs/tech_docs_private/`** (игнорируется git, по смыслу как `.agent/`).
- Назначение: единое место для внутренних материалов владельца рабочей копии; в публичный remote не коммитится. Дальнейший перенос в приватный репозиторий — по желанию владельца.
