## Неделя 2 — детализированные задачи по извлечению показателей и расчёту коэффициентов

Этот документ конкретизирует задачи недели 2 из общего плана и фокусируется на:
- формализации списка **ключевых показателей**;
- определении **формул финансовых коэффициентов**;
- реализации **извлечения показателей и расчёта коэффициентов на бэкенде**;
- обновлении **схем и фронтенда** под новый формат ответа.

### 1. Цели недели 2 (уточнение)

- **Извлечение показателей**: иметь работающий rule-based (regex/паттерны) модуль, который по сырому тексту отдаёт набор ключевых показателей с confidence.
- **Коэффициенты**: зафиксировать и реализовать *минимум 12* финансовых коэффициентов (ликвидность, устойчивость, рентабельность, деловая активность, динамика).
- **Интегральный скоринг (черновой)**: реализовать первый вариант агрегации коэффициентов в скор 0–100 (без претензии на финальную методологию).
- **Интерфейс**: фронтенд должен отображать таблицы показателей и коэффициентов, а также интегральный скор.

#### 2.2. Аналитик 2 (рынок, бизнес-модель)

- **A2.4. Список финансовых коэффициентов и формул**
  - [ ] Утвердить **набор из минимум 12 коэффициентов**, сгруппированных по блокам:
    - Ликвидность:
      - Коэффициент текущей ликвидности = Оборотные активы / Краткосрочные обязательства
      - Коэффициент быстрой ликвидности = (Оборотные активы − Запасы) / Краткосрочные обязательства
    - Финансовая устойчивость:
      - Доля собственного капитала в активах = Собственный капитал / Активы всего
      - Коэффициент финансового рычага (Debt/Equity) = Обязательства всего / Собственный капитал
    - Рентабельность:
      - Валовая рентабельность продаж = (Выручка − Себестоимость) / Выручка
      - Операционная рентабельность = Операционная прибыль / Выручка
      - Рентабельность по EBITDA = EBITDA / Выручка
      - Чистая рентабельность продаж = Чистая прибыль / Выручка
      - ROA (рентабельность активов) = Чистая прибыль / Активы всего
      - ROE (рентабельность собственного капитала) = Чистая прибыль / Собственный капитал
    - Деловая активность:
      - Оборачиваемость активов = Выручка / Активы всего
    - Динамика:
      - Рост выручки год к году = (Выручка_t − Выручка_{t-1}) / Выручка_{t-1}
  - [ ] Для каждого коэффициента зафиксировать:
    - **формулу**;
    - **единицы измерения** (%, x);
    - **категорию** (ликвидность, рентабельность и т.д.).
  - [ ] Описать эти формулы в `docs/ANALYTICS_SPECS.md` в разделе «Финансовые коэффициенты».

- **A2.5. Диапазоны и шкалы для скоринга (черновик)**
  - [ ] Для ключевых коэффициентов предложить **нормальные/критические диапазоны**:
    - например, текущая ликвидность: нормально 1.5–2.5, плохо < 1.0 и т.п.;
    - Debt/Equity: приемлемо до 2–3, выше — рискованно и т.д.
  - [ ] Предложить **грубую схему нормировки** коэффициентов в шкалу 0–1:
    - для коэффициентов “чем больше — тем лучше” (маржи, ROA, ROE, оборачиваемость);
    - для коэффициентов “чем меньше — тем лучше” (Debt/Equity).
  - [ ] Описать эти правила в `docs/ANALYTICS_SPECS.md` в разделе «Скоринговая модель (черновой вариант)».

#### 2.3. Программист 2 (бэкенд)

- **B2.1. Реализация модуля `extractor.py`**
  - [ ] На основе паттернов от аналитиков реализовать:
    - функцию извлечения ключевых показателей `extract_metrics(raw_text: str) -> list[Metric]`;
    - `Metric` как внутреннюю dataclass‑структуру (имя, значение, единица, год, confidence, исходный фрагмент текста).
  - [ ] Логика извлечения (MVP-вариант):
    - построчный проход по тексту;
    - поиск строк, содержащих ключевые фразы/паттерны по каждому показателю;
    - из найденной строки выбирать релевантное числовое значение (например, последнее число в строке);
    - при необходимости — учитывать множители (тыс./млн) и корректировать значение.
  - [ ] Вернуть для каждого показателя:
    - `name` — человеческое название (как в спецификации);
    - `value` — числовое значение;
    - `unit` — единицы измерения;
    - `year` — если удаётся извлечь (иначе `null`);
    - `confidence_score` — грубая оценка уверенности (0–1);
    - `source_fragment` — строка/фрагмент исходного текста.

- **B2.2. Реализация модуля `ratios.py`**
  - [ ] Реализовать функцию `calculate_ratios(metrics: list[Metric]) -> list[Ratio]`, где:
    - `Ratio` — внутренняя структура коэффициента (название, значение, единицы, год, формула, категория).
  - [ ] Поддержать расчёт минимум того набора коэффициентов, который согласует аналитик 2.
  - [ ] Аккуратно обрабатывать случаи:
    - отсутствие нужных показателей (результат `None`);
    - деление на ноль;
    - разные единицы измерения (по возможности нормализовать).

- **B2.3. Реализация модуля `scoring.py`**
  - [ ] Реализовать функцию `calculate_score(ratios: list[Ratio]) -> float | None`, которая:
    - нормирует отдельные коэффициенты в шкалу 0–1 по правилам аналитика 2;
    - агрегирует их (например, простым средним или взвешенной суммой) в скор 0–100;
    - возвращает `None`, если данных недостаточно для осмысленного скоринга.

- **B2.4. Обновление схем `schemas.py` и эндпоинта `/analyze`**
  - [ ] В `backend/app/models/schemas.py`:
    - описать Pydantic‑модели `Metric`, `Ratio` и `AnalyzeResponse` с полями:
      - `metrics: list[Metric]`
      - `ratios: list[Ratio]`
      - `score: float | None`
      - а также поля‑заглушки под NLP/рекомендации/новости (на будущее).
  - [ ] В `backend/app/api/endpoints.py`:
    - обновить логику эндпоинта `/analyze`:
      - извлекать сырой текст из PDF (как в неделе 1);
      - вызывать `extract_metrics` → `calculate_ratios` → `calculate_score`;
      - возвращать заполненный `AnalyzeResponse`.
  - [ ] Добавить базовые **предупреждения** (`warnings`) в ответ (например, если текст не извлечён или мало показателей).

- **B2.5. Подготовка к тестированию**
